The Essential Workflow

1

Working Dir

Where you edit, delete, and create your files locally.

git status
2

Staging Area

Putting files in the "box" before sealing them.

git add .
3

Local Repo

Taking a permanent snapshot of the staged files.

git commit -m
4

Remote Repo

Uploading your history to GitHub to share & backup.

git push

Parallel Universes (Branching)

Never experiment directly on the main branch! Create branches to work safely without affecting production code.

1. List / Create Branches

View all branches, or create a new one without switching to it.

git branch
git branch <name>

2. Switch Branches

Move your working directory to a different timeline. switch is the modern version of checkout.

git switch <name>
PRO-TIP: CREATE & SWITCH SIMULTANEOUSLY
git checkout -b <name>

3. Merge Branches

Combine the completed feature branch back into the main branch. (Ensure you are ON the branch you want to merge INTO).

git merge <name>

Command Hub Cheatsheet

Set your global username:

git config --global user.name "[name]"

Set your global email:

git config --global user.email "[email]"

Initialize a local repository in current folder:

git init

Clone an existing repository from URL:

git clone [url]

Show modified files in working directory (Crucial!):

git status

Add specific file to staging area:

git add [file]

Add ALL modified files to staging area:

git add .

Commit staged files with a message:

git commit -m "[descriptive message]"

Add remote repository URL (connect local to cloud):

git remote add origin [url]

Upload local branch commits to remote repository:

git push origin [branch-name]

Fetch and merge remote commits to your local branch:

git pull origin [branch-name]

Reverse the last commit (keeps changes in working directory):

"OH NO, I COMMITTED TOO EARLY!"
git reset HEAD~1

This removes the commit from history, but your code remains exactly as it was. You can make fixes and commit again.

Show commit history log:

git log --oneline

Unstage a file (keep changes, just remove from the "box"):

git restore --staged [file]

DANGER: Discard all local, uncommitted changes in a file:

git restore [file]
